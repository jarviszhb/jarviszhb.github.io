7.Reverse Integer

Posted on 2019-05-13 | Edited on 2019-05-20 | In leetcode

Given a 32-bit signed integer, reverse digits of an integer. Note: Assume we are dealing with an environment which could only store integers within the 32-bit signed integer range: \([−2^{31}, 2^{31} − 1]\). For the purpose of this problem, assume that your function returns 0 when the reversed integer overflows.

思路分析

  这道题的主要难点是数据的溢出问题，虽然输入的数据一定满足32位整型数的范围，但是经过翻转有可能会溢出。

实现细节

  我的第一个实现方法是先将输入数据中的负数变为正数，但是由于有符号的32位整型数中，负数比正数多一个，因此这个负数变为正数的操作也可能会溢出，因此应当使用长整型。

  在这个实现方法中，我使用了两个长整型的变量，但这实际上是与题目的要求不符的，我对题目的理解是，本题只能使用int型变量，不能使用长整型，因此我对方法1进行了改进。

  在改进的过程中，发现了一个有趣的现象，对于Python语言来说，-91//10的结果是-10，-91%10的结果是9，而在C++中，-91//10的结果是-9，-91%10的结果是-1，这就是说，在用C++实现这道题目的时候，不需要进行负数转换成正数的操作，而如果使用python，可能这步是必不可少的了。

  第二个问题是，其实只需要在循环过程中先对ret变量进行判断，如果运算之后会溢出，则置0，如果不会溢出，则正常进行运算，即使用ret变量与INT_MAX/10进行比较，这样的话，ret变量将永远不会溢出，因此可以使用int型变量。

  x%10/10等于0。
